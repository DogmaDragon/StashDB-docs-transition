# Performer Name

Guidelines relating to a performer's primary name are listed below, roughly sorted by importance. Individual guidelines are also listed in the right-hand sidebar for quick navigation.

For performer guidelines covering different subjects, please see the sidebar to the left.

Guidelines follow a particular format, separating information into a few different sections and sometimes callout boxes. For more details, please see [this section](/faq/#why-are-guidelines-written-that-way) of the FAQ.

Some guidelines will have an "unconfirmed" label inside a green callout box. While the language has not been officially approved yet, editors are still expected to follow and enforce these unconfirmed guidelines. Please see [this section](/faq/#what-are-unconfirmed-guidelines) of the FAQ for more information.

---

## Eligible Names

#### Guideline

{==

**Must be an official adult credit, following same eligibility requirements as performer aliases.**

==}

#### Explanation

A performer's primary name must have been used as an official credit in adult content. In other words, primary names have the same eligibility requirements as performer aliases. While there are notable exceptions for performers who don't use stage names, in short…

!!! danger

    **Legal names and birth names are not allowed anywhere on StashDB**.

#### Guidance

This standard is explained in more detail in the guideline for [Eligible Aliases](../aliases/#eligible-aliases).


## Preferred Names

#### Guideline

{==

**Most popular aliases take precedence, but unique names are preferred in the case of a tie.**

==}

!!! tip ""

    Unconfirmed guideline, subject to change pending formal approval.

#### Explanation

We want to use primary names that communicate as clearly as possible who this performer is. That means we want to choose an alias that can be easily recognized at a glance for as many users as possible, but avoid aliases that could be easily confused with someone else.

In other words, we have to consider both *popularity* and *uniqueness*. Of course, performers often have many aliases to choose from, so it won't always be obvious how best to determine and weigh these two characteristics. Many decisions will ultimately rely on an editor's read of the particular situation. In general, consider popular usage first, then select the most unique alias left to break any ties.

It's also important to note that primary names are only eligible if they've been used as an official adult credit, following the same requirements as [Eligible Aliases](../aliases/#eligible-aliases). Other than that, there aren't any hard rules to follow, but there are a few different factors to take into consideration.

!!! warning "Exceptions"

    - [JAV names](#jav-names) follow their own unique rules regarding spelling and order, please see the linked guideline for more details

#### Guidance

Any aliases that are significantly more popular than the others should take precedence. Sometimes this is obvious, particularly for more famous performers who have only used one alias with any consistency or frequency. If that's the case, then there's no need to think too hard about it. Just make sure to add a [Disambiguation](../disambiguation/) when necessary.

But if there is no obvious answer, consider the following questions: 

- What's the most frequently used alias, both per scene and per studio?
- What's the most commonly used alias on other databases?
- What's the most famous/popular alias among fans?
- What was the performer's preferred alias most recently?

Use these questions to try to narrow down the field of options. If there's a single alias rising above the rest, great. Use that one. If not, uniqueness should be used as the tiebreaker.

Hopefully you've been able to shrink your options down to just a few aliases that seem about as popular as each other. Or maybe you haven't been able to eliminate any options at all yet. Either way, the next step is the same. Any unique aliases that do not require a disambiguation take precedence.

In particular, common single-word names like "Jane" or "John" should be avoided whenever possible, and any that add a last initial like "Jane D." aren't much better. Ideally you want an alias with more than one word, but even that doesn't guarantee uniqueness. A quick search should find any performers using the same name.

But after all of that, sometimes a non-unique name is the only option left. That's fine. Just make sure to add a [Disambiguation](../disambiguation/).

    :white_check_mark: **[Nina Hartley](https://stashdb.org/performers/4368b7c6-ffeb-434d-8714-251e662256e5){:target="_blank"}**
    :   - *Obvious choice, both most popular alias and a unique name*
     
    :white_check_mark: **[Devon](https://stashdb.org/performers/33d7ab5d-cf4f-4de5-a0f8-0d017134dcee){:target="_blank"}**
    :   - *By far the most popular alias, doesn't matter that others are more unique*
    
    :white_check_mark: **[Krystal Boyd](https://stashdb.org/performers/827c414d-de25-4975-87ff-d97d07c919a5){:target="_blank"}**
    :   - *Used in a significant number of scenes, primary name on multiple databases, featured in defunct Twitter bio*
        - *Uniqueness serves as tiebreaker over other equally popular aliases*

    :x: **`Anjelica`**
    :   - *Popular alias, used in the most scenes according to [Indexxx](https://www.indexxx.com/m/anjelica-1), used as primary name in several databases*
        - *Not significantly more popular than `Krystal Boyd`, loses tiebreaker as significantly less unique*

    :x: **`Anjelica Abby` / `Slim Anya` / `Masha Roofkina`**
    :   - *Less commonly used in scenes and by other databases than `Krystal Boyd`*

## JAV Names

#### Guideline

{==

**Western order with simplified Latin characters as default primary name, all other spellings as aliases.**

==}

#### Explanation

JAV performers often have their names ordered in three different ways: Western order (GivenName FamilyName) with Latin characters, Japanese order (FamilyName GivenName) with Latin characters, and Japanese order with Japanese characters. Different sources will prefer different spellings.

For StashDB, we prefer the Western order with Latin characters for all JAV performers as the primary name, matching the formatting of our non-JAV performers. This also makes things simpler for crossover performers who have worked in both Western and JAV productions. Names in the reverse order should always be included as aliases.

To give an example, we use the primary name [Marica Hase](https://stashdb.org/performers/d4f1a54f-ddc7-4f50-a356-d417802cab1c){:target="_blank"} on StashDB. She has done many Western scenes after starting on the JAV side of things, but her primary name is the same regardless of whether she's considered a JAV performer or a Western performer: `Marica Hase`. Alternate spellings `Hase Marica` and `はせまりか` are included as aliases. All three are the same name, just spelled three different ways following three different formats.

!!! warning "Exceptions"

    - An alternate Latin spelling can still take precedence if it's a significantly more popular alias for that performer, please see the [Preferred Names](#preferred-names) guideline above and the **Guidance** section below for more details

#### Guidance

The same [guidelines for picking a primary name](#preferred-names) apply equally to JAV and Western performers. However, JAV performers may also have the same name spelled in various ways following various different conventions for using Latin characters.

Typically, StashDB prefers the use of a simplified Hepburn romanization for primary names which avoids the use of double vowels like `oo` or `uu` as well as diacritic marks like `ō` or `ū`. This is also the same spelling preferred by [X/s List](https://xslist.org/en){:target="_blank"} which is one of the only remaining JAV databases in English.

However, an exception may be allowed on a case-by-case basis if a particular romanization for their name that does not follow this convention is the most commonly used or is known to be the performer's preferred spelling. Again, these are the [same guidelines](#preferred-names) for choosing alternate spellings of Western performer names and they still apply here as well. But due to the difficulty for non-Japanese speakers to determine this for all but the most popular Japanese performers, using the simplified Hepburn spelling as the default should cover most JAV performers without requiring additional research.

As another example, we prefer the spelling of [Yu Shinoda](https://stashdb.org/performers/8465400a-eb3c-4c3f-a78f-09d7541cf27d){:target="_blank"} as the primary name instead of `Yuu Shinoda` or `Yū Shinoda`. If we were creating this performer for the first time, this would be a safe first assumption because it avoids using double vowels or diacritic marks and matches the spelling used on [X\s List](https://xslist.org/en/model/13.html){:target="_blank"}.

If we really wanted to dig deeper (which isn't required for an initial creation and is only possible because she's a popular performer), we'd find different romaji spellings used on [Minnano-AV](http://www.minnano-av.com/actress504478.html){:target="_blank"}, [IMDB](https://www.imdb.com/name/nm6193437/){:target="_blank"}, [Wikidata](https://www.wikidata.org/wiki/Q11603788){:target="_blank"}, and even her own [Instagram](https://www.instagram.com/yuu_shinoda){:target="_blank"}. But since [her agency](http://www.t-powers.co.jp/official/talent/%e7%af%a0%e7%94%b0%e3%82%86%e3%81%86/){:target="_blank"}, [Twitter](https://twitter.com/yu_shinoda){:target="_blank"}, [IAFD](https://www.iafd.com/person.rme/perfid=yushinoda/gender=f/yu-shinoda.htm){:target="_blank"}, [FreeOnes](https://www.freeones.com/yu-shinoda){:target="_blank"}, and [Babepedia](https://www.babepedia.com/babe/Yu_Shinoda){:target="_blank"} all use `Yu Shinoda`, that remains the preferred option because it appears to be her own preferred romanization as well as the most commonly used spelling for English language databases. All other spellings that can be found should be used as aliases instead.

!!! quote "Examples"

    :white_check_mark: **[Marica Hase](https://stashdb.org/performers/d4f1a54f-ddc7-4f50-a356-d417802cab1c){:target="_blank"}**
    :   - *Most popular name and spelling, Western order, and simplified Hepburn romanization*
        - *Preferred name does not change whether she's considered primarily a JAV or Western performer*

    :x: **`はせまりか`**
    :   - *Does not use Latin characters*

    :white_check_mark: **[Yu Shinoda](https://stashdb.org/performers/8465400a-eb3c-4c3f-a78f-09d7541cf27d){:target="_blank"}**
    :   - *Most popular name and spelling, Western order, and simplified Hepburn romanization*

    :x: **`Shinoda Yu`**
    :   - *Uses Japanese order (FamilyName GivenName) instead of Western order (GivenName FamilyName)*

    :x: **`Yuu Shinoda` / `Yū Shinoda`**
    :   - *Uses double vowels (`uu`) or diacritics (`ū`) instead of simplified Hepburn romanization (`u`)*
        - *Neither are more popular aliases than `Yu Shinoda`*

## Diacritics

#### Guideline

{==

**Diacritics are allowed in the primary name, but the name without diacritics must be included as an alias.**

==}

#### Explanation

Diacritics are the marks added above or below a letter to indicate a difference in pronunciation. Two common examples are the acute ( ◌́ ) and the grave ( ◌̀ ). The primary name of a performer may include any of these diacritical marks but the same name without any diacritics will need to be included as an alias. This makes it easier to find them in search results.

!!! warning "Exceptions"

    - [JAV Names](#jav-names) follow their own rules for diacritics, please see the linked guideline for more details

#### Guidance

Please be aware that the same considerations explained in [Preferred Names](#preferred-names) still apply here. A name without diacritical marks may still be preferrable as the primary if it's by far the most commonly used spelling for that performer, for example. If that's the case, the version with diacritics should still be included as an alias, assuming it's [eligible](../aliases/#eligible-aliases).

!!! quote "Examples"

    :white_check_mark: **[Vitória Vonteese](https://stashdb.org/performers/fbd10ce7-3209-4788-b84f-3a2ec1b19326){:target="_blank"}** 
    :   - *Diacritic (ó) in her primary name, `Vitoria Vonteese` (no diacritics) included as an alias*

    :white_check_mark: **[Yu Shinoda](https://stashdb.org/performers/8465400a-eb3c-4c3f-a78f-09d7541cf27d){:target="_blank"}** 
    :   - *No diacritic (`u`) in her primary name, `Yū Shinoda` (`ū`) included as an alias*

    :x: **`Yū Shinoda`**
    :   - *Spelling with diacritic (`ū`) is less popular than `Yu Shinoda`, not a [preferred primary name](#preferred-names)*
        - *Does not follow simplified Hepburn romanization, not a [preferred JAV spelling](#jav-names)*
